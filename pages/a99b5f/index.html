# 5. 固定参数的易解性(FIXED PARAMETER TRACTABILITY)

我们重新回顾GFDs的三个关键问题:

  1. 满足性问题(Satisfiable)

    是否存在一个图数据, 能满足所有GFDs.

    1. GFDs不存在冲突.
    2. 至少一个GFD是可以用在非空图数据上的.

  2. 蕴含问题(Implication)

    判断一个GFD是否能被一个GFDs的集合推导出来. 对于判断多余的GFDs来说很有必要.

    比如:

    和, 明显第二个可以推出前一个, 所以第一个是多余的.

  3. 验证问题(Validation)

    判断一个GFDs集合能否满足一个特定的图数据

    和满足性问题的区别在于此处的是给定的, 而满足性问题里面的是任意的, 存在即可.

satisfiablilty, implication和validation三个问题分别是coNP-complete, NP-complete以及coNP-complete问题

Open World Assumption(OWA) 定义

OWA指出，不存在的数据不能作为知识库中的反例。

  1. 对于正的GFD，其支持度量化了存在并符合的实体
  2. 对于负GFD ，其支持度由正GFD的支持度所决定; 也就是说, 负GFDs描述了观察世界中“不存在”的案例; 未知数据对负GFDs的发现没有影响.

# 6.3 The Discovery Problems

我们将GFDs的挖掘问题表述为:

  • 输入: 一个图, 一个自然数和一个支持度阈值
  • 输出: 一个, k-bounded最小GFDs的覆盖集

验证问题和蕴含问题被包含在GFD的挖掘过程中, 对应步骤: 检查和计算一个k-bounded GFDs的覆盖集

我们使用这个参数来平衡挖掘过程和对GFDs解释的复杂性. 因为显然地:

  1. 如果GFDs拥有太多属性则不太可能频繁, 并且也很难跟终端用户进行解释
  2. 过大的时候计算复杂度过高, 验证问题和蕴含问题是当是固定的时候是PTIME的
